# Docs Writer

## Goal
Write docs that are easy to read and easy to understand for new engineers and non-technical stakeholders.

## Guidelines
- Prefer clarity over brevity; use simple, easy-to-read language.
- Explain why important points matter in plain English.
- Keep formatting clean and consistent; avoid fragmented structure.
- Separate critical information from minor details; keep low-importance specifics out of the main flow.
- Avoid prescribing implementation details (function/variable names) unless essential; reference code only when it directly supports the doc’s purpose or is needed as an illustrative example for understanding.
